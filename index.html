HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Hypertext Markup Language (HTML) is one of the three cornerstone languages used in web development.
  • HTML is used to create the structure of a webpage.
  • It is made up of elements which work together to provide content and the basic framework for all pages on the web.
  • Elements are used to add content to pages.
  • There are multiple types of elements including: text, image, header, and list.
  • Elements are also used to divide the content on the page and to describe the content to browsers and other developers.
  • Common types of elements used to divide content are: meta-related, content division, and semantic elements.
  • The "Head" elements contains information or metadata that helps the browser to render the page correctly.
  • The "Body" element contains the content that is show to the user.
  • Any content we want the end user to see and interact with must be placed inside the "Body" element.
  • The "Doctype html" at the top of the page is considered a "declaration."
  • "Declarations" are used to tell the browser what document type it should expect when opened.
  • "Doctype" is not case sensitive, but HTML elements are.
  • "Metadata" is data about the webpage that the browser uses to effectively serve our webpage to users.
  • "Metadata" is not typically seen by users.
  • The "style" element defines the CSS styles associated with a website.
  • The "base" element defines the URL for a webpage.
  • The "meta" element defines metadata such as the character set, description, keywords, author, and viewport.
  • The "script" element embeds client-side scripts in the HTML document.
  • The "charset" or character set element, sets our character encoding.
  • When the attribute is set to "lang="en". This lets us know the page will be in English.
  • The "http-equiv" element sets an HTTP header for content.
  • The "viewport" element sets the default view for our webpage.
  • The "section" element divides content up into more organized sections based on thematic content.
  • The "class" attribute is used to define a
  • The "id" attribute is a way to identify elements and select them.
  • Using the "ul" tag, creates an "unordered list" of items. This is a bulleted list.
  • Using the "ol" tag, creates an "ordered list". This is a numbered list.
  • Using the "li" tag, creates a "list item" for an unordered or ordered list.

CSS

  • CSS stands for "Cascading Style Sheets".
  • CSS defines how webpage content is presented to users.
  • For example it defines things such as: webpage colors, aesthetics, visual, layout, and organization.
  • The three ways to style a webpage using CSS are: inline CSS, internal CSS, and external CSS.
  • When using an external CSS style sheet. You use a link element to link the HTML page to a separate CSS file.
  • When using "external CSS", the link is placed between the HTML files "head" elements.
  • A "margin" indicates how much space we want around the outside of an element.
  • A "padding" indicates how much space we want around the content inside an element.
  • The "." that precedes the "card" designates this as a class selector.
  • Any HTML element assigned to the class will gain whatever property assignments you defined in your CSS code.
  • The "bow-shadow" CSS property adds shadow effects around the elements frame.
  • Each box consists of margins outside the element, and the element's "borders, padding, and content." This is sometimes referred to as the "CSS box model."
  • When trying to center something on a page, such as text or an image. It is best to use the max-width and min-width commands in addition to the width command if you want to really get more precise.

Git

  • The "git status": command checks what branch we are currently on.
  • The "git checkout -b branch-name": command creates a new branch and switches to it.
  • The "git add -A": command to commit all files in the working branch.
  • The "-m": flag to associate a message with our commit.
  • The git branch -b branch name command creates a branch, but does not check you out to it.

Steps to Sync Local files with GitHub


  1. When you are on or in the main branch locally. We have a requirement to work on a feature or separate branch.
  2. Checkout to a new branch with the git checkout -b branch name command.
    1. This will create and checkout to the new branch name.
  3. Do the commit workflow as needed in this stage. This includes pushing those commits to GitHub.
      4. This means do either git add . or git add -A.
  4. Sync with GitHub with the git push command.
    1. Warning this will prompt you with the following suggestion example:
    1. push --set-upstream origin "branch name".
    2. Now we will have a synced connection between GitHub and our local branch.
  5. Work on your files/directories locally.
  6. Create a Pull Request (PR) in GitHub
  7. Then do a Merge Pull Request.
  8. On the local device. If we are still not in main branch, we need to checkout.
    1. To checkout to the main branch, use the git checkout main command.
  9. Then we need to bring all the changes living in GitHub remote/main branch to our local machine using the git pull command.
  10. When you finish, rinse and repeat as needed

Javascript

  • A file extensions of ".js" indicates the file as a "JavaScript" file.
  • A "variable" is a named container that allows us to store data in our code.
  • Before previous changes to Javascript you would use the keyword "var" to create a "variable in your code.
  • However, you now use "const" to create a "variable".
  • In JavaScript a "string" is surrounded by quotation marks.
  • The "console.log()" method, allows you to output a message to the web console by adding an "argument".
  • The "argument" is put inside the parenthesis that follow "console.log."
  • A "JavaScript file is linked inside the "body" element. Whereas, CSS is linked in the "head" element.
  • A group of related data in JavaScript is known as an "array".
  • "Arrays" are especially useful when you want to perform a task on multiple pieces of data at once.
  • A for loop uses the predictable pattern of "indices" to perform a task on all items in an array, by allowing a single code block to be executed over and over.
  • A function's name is followed by "()".
  • The code for a function is also written inside curly brackets or "{}".
  • Control flow is the order in which a computer executes code in a script.
  • An object in JavaScript is a group of key value pairs.
  • In a key value pair The key is the part before the ":" symbol on the left. The part following the ":" symbol is the value.
  • In the DevTools of a browser. The "{}" signify an object on the Console tab.
  • The "[]" on a web broswers Console tab indicate the use of an array. This is also the case outside of the browser.
  • An array can also be considered as a list of items that can be added to if need be.
  • An object is an example of one item that can have multiple properties to it.
