Git

Steps to Sync Local files with GitHub

  1. When you are on or in the main branch locally. We have a requirement to work on a feature or separate branch.
  2. Checkout to a new branch with the git checkout -b branch name command.
    1. This will create and checkout to the new branch name.
  3. Do the commit workflow as needed in this stage. This includes pushing those commits to GitHub.
      4. This means do either git add . or git add -A.
  4. Sync with GitHub with the git push command.
    1. Warning this will prompt you with the following suggestion example:
    1. push --set-upstream origin "branch name".
    2. Now we will have a synced connection between GitHub and our local branch.
  5. Work on your files/directories locally.
  6. Create a Pull Request (PR) in GitHub
  7. Then do a Merge Pull Request.
  8. On the local device. If we are still not in main branch, we need to checkout.
    1. To checkout to the main branch, use the git checkout main command.
  9. Then we need to bring all the changes living in GitHub remote/main branch to our local machine using the git pull command.
  10. When you finish, rinse and repeat as needed.

Javascript

  • JavaScript is a loosely-typed langugage and can be prone to errors in your code.
  • A file extensions of ".js" indicates the file as a "JavaScript" file.
  • A "variable" is a named container that allows us to store data in our code.
  • Before previous changes to Javascript you would use the keyword "var" to create a "variable" in your code.
  • However, you now use "const" or "let" to create a "variable".
  • In JavaScript a string is surrounded by "".
  • The "console.log()" method, allows you to output a message to the browsers Console tab by adding an "argument".
  • An "argument" is put inside the parenthesis.
  • In JavaScript you can look inside of an object for other properties using a "."
  • In the following example, "document" contains a "property" of body. body contains a property of style. style contains a property of backgroundColor.
    1. document.body.style.backgroundColor
  • A JavaScript file is linked inside the "body" element. Whereas, CSS is linked in the "head" element.
  • A group of related data in JavaScript is known as an "array".
  • Arrays are especially useful when you want to perform a task on multiple pieces of data at once.
  • A for loop uses the predictable pattern of "indices" to perform a task on all items in an array, by allowing a single code block to be executed over and over.
  • A function's name is followed by "( )".
  • The code for a function is also written inside curly brackets or "{ }".
  • Control flow is the order in which a computer executes code in a script.
  • An object in JavaScript is a group of key value pairs.
  • In a key value pair The key is the part before the ":" symbol on the left. The part following the ":" symbol is the value.
  • In the DevTools of a browser. The "{ }" signify an object on the Console tab.
  • The "[ ]" on a web broswers Console tab indicate the use of an array. This is also the case outside of the browser.
  • An array can also be considered as a list of items that can be added to if need be.
  • An object is an example of one item that can have multiple properties to it.
  • The .length command specifies the length of an array.
  • If for example you write let studentName = Tonya. Then studentName is the variable name and Tonya is the value.
  • Undefined means the variable has not been defined or assigned to anything.
  • Null means the variable was defined as Null.
  • Both Null and Undefined are different ways to say nothing.
  • The modulus, (%) operator is used to find if a number has a remainder. Which in turn can help you determine if the number is odd or even.
  • The (==) helps determine if the variables are equal in value, but not by type.
  • The (!=) helps determine if the variables are not equal in value.
  • The (===) checks to see if the two variables are equal in type and value.
  • The (!==) checks to see if the values and types are not equal.
  • The && operator checks that both expressions are true in value.
  • If you create a while loop everytime the loop ends all previous variables inside the loop are refreshed or thrown out. In this situation you can use const instead of let.
  • You can only use const instead of let inside of a while loop when you are NOT changing the value of the variable anywhere else outside of the while loop OR are NOT reassigning the variable anywhere else.
  • JSON.parse is a static method that parses a JSON string, constructing the JavaScript value or object described by the string.
  • An optional reviver function can be provided to perform a transformation on the resulting object before it is returned. In other words, it turns the string back into an object.
  • JSON stands for = JavaScript Object Notation.
  • Always remember stringify to set and parse to get.
  • Always use JSON.stringify when doing localStorage.setItem first and then JSON.parse to localStorage.getItem after.
  • When you setItem it takes a key and a value, while getItem takes just the key.
  • localStorage.clear clears all data from localStorage.

TypeScript

  • TypeScript is an open-sourced and strongly-typed syntactic superset of JavaScript.
  • The Union operator or "|" is used when a value can be more than one single type. Such as when a property can be a "string" or a "number"
  • the "void" in TypeScript serves as a primitive type and is a continuation subtype of undefined.
  • A primitive in TypeScript or programming in general, is a basic data type used to represent simple data or values. In other words, the building blocks or simplest form of data that a programming language can understand.
  • TypeScript interfaces and classes should be PascalCase. Meaning the first letter of each word is capitalized and the rest are lowercase.
  • It is generally best practice to first define the interface and its properties or key value pairs.
  • If you do not know what you are going to define within an interface. Then it is best to create an object first.
  • The only time you would an "I" at the begining of an interface, is when their naming might conflict with TypeScript classes.
  • A constructor function are functions that create objects.
  • The new keyword creates a "new" empty object.
  • The new keyword then binds the newly created object to the prototype of the constructing function.
  • When you make a property or variable private. Then that property or variable is only available in that class.
  • You can destructure an array to turn its values into distinct variables.
  • You can also destructure an objects properties into distinct variables as well.

Servers and APIs

  • You can use Node.js to create new files on your computers hard drive, listen for incoming network requests, and more.
  • REST stands for Representational State Transfer.
  • REST is an architectural patten that...
  • Servers that hold data that we can request are known as server-side APIs.
  • API stands for Application Programming Interface.
  • Application refers to any software with a distinct function.
  • Interface can be thought of as a contract of service between two applications.
    • The interface contract defines how the two endpoints communicate using requests and responses.
  • APIs can be either simple or complex.
  • The CRUD acronym stands for Create, Read, Update, and Delete.
  • GET is used to read data. In other words, the client can request data but NOT change its state.
  • POST is used to create data.
  • PUT is used to update data.
  • DELETE is used to delete data.
  • The "listen()" method is used to start a web server and listen to the connections on the specified host and port.
  • In server code, anytime you see code prefixed by a ":". This means that it is a route parameter variable.
  • A route parameter variable is dynamic and be swapped out.
  • The find() method returns the value of the first element in the array where predicate is true, and undefined otherwise.
  • In server terms predicate signifies a logical condition that evaluates to a Boolean value (true or false).
  • The fetch() API function returns a GET request by default.
  • A wild card is indicated in server code by the "*" symbol.
  • The wild card has to be the last route in your list of API calls.
  • Any code that follows a ? is known as a query.
  • The HTTP Status Codes 400 series indicates that there is an error client-side.
  • The HTTP Status Codes 500 series indicates there is an error server-side.
  • UUID stands for Universally Unique Identifier.
  • A client sends a "request" to a server asking for some kind of information. Upon receiving the request the server(s) send a "response" back to the client.

SQL

  • SQL stands for Structured Query Language
  • A schema is a collection of database objects that are logically connected and organized.
  • Schema can also refer to the structure or (blueprint) of data in a database and how the elements relate to each other.
  • The following code is how you login to PostgreSQL in the terminal. After entering this command you would type in your PostgreSQL password. 
    1. psql -U postgres
  • The following code signifies an input command: 
    1. \i
  • The following file type is used to hold mock data as you are building your database. 
    1. schema.sql
  • The following command is an example of how to create a database. 
    1. CREATE DATABASE (database name).db
  • It is best practice for many SQL commands to be typed in all capital letters. The following code is one such example. 
    1. CREATE DATABASE
  • The following code is used to show a list in the terminal of all databases on your local device. 
    1. /l
  • The following code stands for Variable Character: 
    1. VARCHAR
  • VARCHAR is basically a string of characters.
  • Whatever string number is set inside "VARCHAR" is the maximum amount of characters allocated for that table.
  • In PostgreSQL ACID stands for Atomicity, Consistency, Isolation, and Durability.
